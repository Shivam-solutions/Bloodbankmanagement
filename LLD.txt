2.3 API Design
Endpoints
User

POST /register: Register a new user
POST /login: User login
PUT /updateProfile: Update user profile
Donor

POST /donate: Register a new blood donation
GET /donationHistory: Get donation history
Inventory

POST /addBlood: Add new blood to inventory
PUT /updateBlood: Update blood details
GET /checkAvailability: Check blood availability
Request

POST /createRequest: Create a new blood request
PUT /updateStatus: Update request status
GET /trackRequest: Track the status of a request
2.4 Sequence Diagrams
User Registration
User submits registration form.
System validates input data.
System creates a new user record in the database.
System sends a confirmation email to the user.
Blood Donation
Donor submits blood donation form.
System validates donor eligibility.
System updates the blood inventory.
System updates the donor's donation history.
Blood Request
Hospital/patient submits a blood request.
System checks blood availability.
System updates the request status.
System notifies the hospital/patient of the request status.
